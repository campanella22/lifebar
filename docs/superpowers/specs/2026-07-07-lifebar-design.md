# LifeBar 設計ドキュメント

作成日: 2026-07-07 / ステータス: レビュー待ち

## 1. コンセプト

Mac のメニューバーに住むドット絵の男を、勉強時間で育てる常駐タイマーアプリ。
勉強すると男は筋トレでムキムキになり、稼ぎ始め、彼女ができ、家族が増え、孫に囲まれる。
サボると彼女に振られ、会社が倒産し、妻が家を出て行く。

- **目的**: GitHub で「なにこれw」とスターされるキャッチーさが第一。タイマーとして毎日使える最低限の実用性は担保。
- **仮タイトル**: `LifeBar`（メニューバー × 人生ゲージのダブルミーニング。公開前に GitHub 上の名前衝突を確認して最終決定）
- **プラットフォーム**: macOS 14+ / Swift / 依存ライブラリゼロ

## 2. ゲームデザイン

### 2.1 パラメータと成長

筋肉・金・愛の 3 パラメータ。各パラメータは XP 制で Lv0〜Lv4 の 5 段階。

- **勉強 1 分 = 1 XP**。タイマー開始時に投資先（筋肉/金/愛のどれか 1 つ）を選び、そのセッションの XP は全額そこに入る。

| Lv | 必要XP（累計） | 筋肉 💪 | 金 💰 | 愛 ❤️ |
|----|---------------|---------|-------|-------|
| 0 | 0 | もやし | 無職・ボロアパート | 独身 |
| 1 | 120（2時間） | 普通の体 | バイト | 彼女ができる |
| 2 | 600（10時間） | 引き締まる | 就職（スーツ） | 結婚 |
| 3 | 1,500（25時間） | マッチョ | 起業・社長 | 子供が生まれる |
| 4 | 3,000（50時間） | ゴリマッチョ | 豪邸 | 孫に囲まれる |

3 パラメータ全 MAX で計 150 時間（1日2時間で約2.5ヶ月）。

### 2.2 減衰（たまごっち性）

- 各パラメータは **1 時間に 1 XP（1日 24 XP）ずつ常時減衰**（下限 0）。
- タイマー稼働中は**全パラメータの減衰が停止**する。
- 目安: **1日合計 72 分の勉強で人生の現状維持**。
- アプリ非起動中の減衰は、次回起動時に前回タイムスタンプとの差分から遅延計算する（常駐バックグラウンド処理は持たない）。

### 2.3 レベル判定とヒステリシス（バタつき防止）

- **レベルアップ**: XP が閾値に到達したら即時昇格。
- **レベルダウン**: XP が「現レベル閾値 × 0.9」を下回って初めて転落（落ちにくく、戻りやすく）。
- このため**現在レベルは保存状態**として持つ（XP から一意には導出できない）。

### 2.4 イベント

レベル変動時にドット絵イベントカード＋ macOS 通知で演出する。

| Lv変動 | 筋肉 💪 | 金 💰 | 愛 ❤️ |
|--------|---------|-------|-------|
| 昇格 | 体が締まってきた！/ マッチョに！/ 等 | バイト採用！/ 就職！/ 起業！/ 豪邸購入！ | 彼女ができた！/ 結婚！/ 子供誕生！/ 孫が来た！ |
| 転落 | リバウンド… / 腹が出てきた… / 三段腹… / もやしに戻った… | バイトをクビ… / 会社が倒産… / 借金取りが来た… / 豪邸差し押さえ… | 彼女に振られた… / 妻が家を出て行った… / 子供が口をきいてくれない… / 孫が来なくなった… |

- 転落が近い時（減衰でレベルダウンまで残り 24 時間を切った時）に**1 回だけ**警告通知:「⚠️ 彼女が別れを切り出しそうです」。サボり防止の実用機能を兼ねる。

### 2.5 称号

3 パラメータのレベル組み合わせで称号を決めてシーン下に表示する。ここが最大のネタどころ。

例: 「ムキムキ無職」（筋肉Lv3+・金Lv0）、「痩せた社長」（筋肉Lv0・金Lv3+）、「愛だけはある男」（愛Lv2+・他Lv0）、「人生の勝者」（全Lv4）。
称号は判定ルールのテーブルとして `TitleRule.swift` に定義し、ユニットテストを書く。

### 2.6 エンディングと周回

- **勝者エンディング**: 全パラメータ Lv4 →「人生の勝者」演出 → 殿堂入り（達成日・所要時間・周回数を記録）→ 新しい人生を開始（XP リセット、周回数 +1）。
- **どん底エンディング**: 全パラメータ XP=0 の状態が 72 時間続く → 「男は田舎に帰った」演出 → 新しい人生を開始。

### 2.7 バランス定数

上記の数値（閾値・減衰率・ヒステリシス係数・警告タイミング・どん底判定時間）はすべて `Core/Balance.swift` に集約し、後から一括調整できるようにする。

## 3. UI/UX

### 3.1 メニューバー（常時表示）

- 高さ 18px 前後の**ドット絵の男だけ**が常駐。文字なし。
- 体型は筋肉 Lv を反映。
- アニメ状態: 勉強中は投資先別アニメ（筋トレ / 札束数え / ハート）、通常は待機、衰弱時（いずれかのパラメータが転落 24 時間前）はうなだれる。各状態 2〜4 コマ。
- タイマー稼働中のみ隣に経過時間を表示（設定で OFF 可、デフォルト ON）。

### 3.2 ポップオーバー（クリックで表示）

**メイン画面は「絵・ゲージ・開始ボタン」の 3 要素だけ**。他はすべて詳細画面に格納する。

```
┌──────────────────────────┐
│   ドット絵の人生シーン（大）        │ ← 背景=金Lv / 男の体型=筋肉Lv / 同伴者=愛Lv
│   「ムキムキ無職」← 称号          │    README のスクショになる一枚絵
│  💪 ▓▓▓▓▓░░░░░ Lv2        │
│  💰 ▓░░░░░░░░░ Lv0        │ ← ゲージ 3 本。数字なし、バーと Lv のみ
│  ❤️ ▓▓▓░░░░░░░ Lv1        │
│   [💪]  [💰]  [❤️]        │ ← 投資先を選んでワンクリック開始
│    今日どれに人生を賭ける？       │    稼働中は「⏱ 42:15 ■停止」に変化
│              ⋯（詳細）      │
└──────────────────────────┘
```

- シーンはレイヤー合成: 背景（金Lv: ボロアパート→豪邸）＋男（筋肉Lv 体型）＋同伴者（愛Lv 累積: Lv1=彼女、Lv2=妻、Lv3=妻+子供、Lv4=妻+子供+孫）。
- 詳細画面（⋯から開く）: 今日の合計 / 累計時間 / イベントログ / 殿堂 / 設定（ログイン時起動・経過時間表示・通知）/ 人生リセット。
- イベントカード: ポップオーバー上にオーバーレイ演出。専用イラストはエンディング 2 枚のみ。他のカードは既存キャラ絵＋テキストの合成で済ませ、絵の枚数を増やさない。

### 3.3 通知

- 昇格・転落イベント時と転落警告時のみ。それ以外は通知しない。

## 4. アーキテクチャ

```
LifeBar/
├── LifeBarApp.swift             # エントリポイント
├── Core/                        # 純粋ロジック層（UI から完全分離、全部テスト対象）
│   ├── Balance.swift            # バランス定数の一元管理
│   ├── LifeState.swift          # 3パラメータXP・現在レベル・タイムスタンプ・周回などのモデル
│   ├── LifeEngine.swift         # XP加算・減衰・レベル判定・イベント発火（純粋関数）
│   └── TitleRule.swift          # 称号判定テーブル
├── UI/
│   ├── MenuBarController.swift  # NSStatusItem＋コマ送りアニメ（Timer駆動）
│   ├── PopoverView.swift        # メイン画面（SwiftUI）
│   ├── DetailView.swift         # 詳細画面
│   └── EventCardView.swift      # イベント演出
├── Persistence/
│   └── Store.swift              # Application Support の JSON 1ファイル読み書き
├── Assets/sprites/              # 生成済み PNG（コミットする）
├── tools/generate_sprites.py    # テキストグリッド＋パレット → PNG 生成スクリプト
└── Tests/LifeEngineTests.swift
```

### 4.1 設計判断

- **減衰の遅延計算**: `LifeState` に `lastTickAt` を持ち、UI の定期 tick（1 分間隔）とアプリ起動時に `LifeEngine.tick(state, now) -> (newState, [Event])` を呼ぶ。純粋関数なので経過時間ゼロから数日分まで同一ロジックでテスト可能。
- **メニューバーは AppKit**: コマ送りアニメの確実性のため `NSStatusItem` を直接使う（RunCat 方式）。ポップオーバー内は SwiftUI。
- **スプライトのレイヤー合成**: 背景 5 種（金）＋男の体型 5 種 ×アニメ状態（筋肉）＋同伴者 4 種（愛）の**足し算**で管理し、組み合わせ爆発（掛け算）を避ける。
- **生成スクリプトで体型をパラメータ化**: 男の 5 体型は 1 テンプレートから肩幅・腕の太さをパラメータで変えて生成し、手描き相当の枚数を増やさない。PNG は @1x/@2x でコミットし、後から高品質素材への差し替えは PNG の置き換えだけで済む。
- **永続化は JSON 1 ファイル**: `~/Library/Application Support/LifeBar/state.json`。変更時と終了時に保存。スキーマに `version` を持たせ将来の移行に備える。

### 4.2 保存データのスキーマ（v1）

```json
{
  "version": 1,
  "params": {
    "muscle": {"xp": 0, "level": 0},
    "money":  {"xp": 0, "level": 0},
    "love":   {"xp": 0, "level": 0}
  },
  "lastTickAt": "2026-07-07T09:00:00Z",
  "session": {"target": "muscle", "startedAt": "2026-07-07T09:00:00Z"},
  "run": 1,
  "hallOfFame": [],
  "eventLog": [],
  "settings": {"showElapsed": true, "launchAtLogin": false}
}
```

`session` は非稼働時 null。`level` を保存するのはヒステリシスのため（§2.3）。

### 4.3 デバッグ

- 起動引数 `-timeScale 100` で時間を早回しして減衰・イベント・エンディングを目視確認できるようにする。
- 起動引数 `-demoState <preset>` で「ムキムキ無職」等の代表状態を再現。

## 5. テスト

- `LifeEngine` のユニットテスト（XCTest）: XP 加算 / 減衰計算（数日分の差分含む）/ 昇格・転落のヒステリシス / 警告発火が 1 回だけであること / エンディング条件 / 周回リセット。
- `TitleRule` のテーブルテスト。
- UI は手動確認（`-timeScale` と `-demoState` を使用）。

## 6. GitHub 公開・配布

- **README**: 冒頭に人生の栄枯盛衰 GIF。日本語＋英語併記。
- **ライセンス**: MIT（アセットも自前生成なので完全クリーン）。
- **配布**: GitHub Releases に .app の zip（ad-hoc 署名。初回起動は右クリック→開く。手順を README に記載）。スターが付いたら Homebrew cask を検討。
- **CI**: GitHub Actions（macos-14 runner）で build + test。

## 7. スコープ外（YAGNI）

- ポモドーロ・目標設定・詳細統計グラフ
- オンライン同期・ランキング・対人要素
- アンチチート（放置で XP 稼ぎ等は性善説に任せる）
- Windows / Linux 対応
- キャラの性別・見た目カスタマイズ（まずは「男」一本。ウケたら検討）
